自從經手動輒千份以上的資料，要手動點選翻閱是一件費時的工作。即使MacOS和Win11都有支援的官方查詢，但都沒有自己熟悉檔案系統來得有效率。在Mac系統上支援終端機(Terminal)，連動Linux語法有很多好處。例如find或者du都是常用的檔案管理技巧。

一鍵查詢目前所有資料夾的大小，並整合排序
```zsh
du -shc ./* |sort -h
```

或是一鍵模糊搜尋檔案名稱
```zsh
find . -type f -name "*我是關鍵字*" 
```

或是一鍵模糊搜尋資料夾名稱
```zsh
find . -type d -name "*我是關鍵字*" 
```

針對找到的批量資料進行操作，例如移動上千檔案
```zsh
find . -type f -name "*我是關鍵字*" -exec mv {} 我是目標資料夾名稱/ \;
```

這些功能如何在win11上面運作呢？透過powershell可以有類似的指令做法。我用gemini協助以powershell寫成一個可執行的腳本，即使在離線不連網的情況也可以進行批量資料整理

### 模糊快速查詢指令

```powershell
Get-ChildItem -Path . -Filter "*我是關鍵字*" -File -Recurse
```
>-Path .：從當前目錄開始搜尋（. 代表目前路徑）。
-Filter "*我是關鍵字*"：搜尋檔名包含該關鍵字的檔案。這比使用 -Include 效率更高，因為它是在系統讀取檔案時就進行過濾。
-File：只尋找「檔案」，排除資料夾（相當於 find 的 -type f）。
-Recurse：遞迴搜尋所有子目錄。

### 模糊查詢後進行批量移動腳本
```powershell
# 取得關鍵字並自動補全萬用字元
Write-Host "`n[3/3] 請輸入搜尋關鍵字 (例如 .png 或 2022):" -ForegroundColor Yellow
$rawPattern = $Host.UI.ReadLine().Trim()
if ($rawPattern -notlike "**") {
    $searchPattern = "*$rawPattern*"
} else {
    $searchPattern = $rawPattern
}

# --- 2. 路徑合法性檢查 ---
if (!(Test-Path -Path $source)) { 
    Write-Host "`n[錯誤] 找不到來源路徑！請檢查路徑是否正確。" -ForegroundColor Red
    Write-Host "偵測到的路徑：$source" -ForegroundColor Gray
    Write-Host "按 Enter 鍵結束..."
    $null = $Host.UI.ReadLine()
    exit 
}

# 若目標資料夾不存在則自動建立
if (!(Test-Path -Path $dest)) { 
    New-Item -ItemType Directory -Path $dest -Force | Out-Null 
    Write-Host "`n[提示] 已建立目標資料夾。" -ForegroundColor Gray
}

# --- 3. 掃描檔案與預覽 (Dry Run) ---
Write-Host "`n正在掃描符合 '$searchPattern' 的項目..." -ForegroundColor Cyan

# 取得項目 (排除目標路徑本身，避免迴圈搬移)
$items = Get-ChildItem -Path $source -Recurse | Where-Object { 
    $_.Name -like $searchPattern -and $_.FullName -notlike "$dest*" 
}

if ($null -eq $items -or $items.Count -eq 0) {
    Write-Host "找不到符合的項目。請確認關鍵字是否正確。" -ForegroundColor Yellow
    Write-Host "`n按 Enter 鍵結束..."
    $null = $Host.UI.ReadLine()
    exit
}

Write-Host "`n--- 預計搬移清單 (共 $($items.Count) 個項目) ---" -ForegroundColor Green
$items | Select-Object @{Name="名稱"; Expression={$_.Name}}, @{Name="原始位置"; Expression={$_.FullName}} | Format-Table -AutoSize

# --- 4. 執行確認與搬移邏輯 ---
Write-Host "是否確認執行搬移？(Y/N): " -NoNewline -ForegroundColor White
$confirm = $Host.UI.ReadLine().Trim()

if ($confirm -eq "Y" -or $confirm -eq "y") {
    $successCount = 0
    $skipCount = 0

    foreach ($item in $items) {
        $targetPath = Join-Path $dest $item.Name
        
        # 安全檢查：若目標已存在則忽略，避免覆蓋
        if (Test-Path -Path $targetPath) {
            Write-Host "[跳過] 檔案已存在：$($item.Name)" -ForegroundColor Yellow
            $skipCount++
        } else {
            try {
                Move-Item -Path $item.FullName -Destination $dest -ErrorAction Stop
                Write-Host "[成功] 搬移：$($item.Name)"
                $successCount++
            } catch {
                Write-Host "[失敗] 無法搬移 $($item.Name)：$($_.Exception.Message)" -ForegroundColor Red
            }
        }
    }
    Write-Host "`n任務完成！" -ForegroundColor Cyan
    Write-Host "成功: $successCount" -ForegroundColor Green
    Write-Host "跳過: $skipCount" -ForegroundColor Yellow
} else {
    Write-Host "`n操作已取消。" -ForegroundColor Gray
}

# --- 5. 結束程式 (跨平台相容) ---
Write-Host "`n[結束] 請按 Enter 鍵關閉視窗..." -ForegroundColor Gray
$null = $Host.UI.ReadLine()
```

### 實際執行細節

在powershell上面要執行ps1腳本，需要先開啟權限。為了不被惡意程式干擾，我使用一次性開啟powershell的方式啟用權限，透過以下程式碼開啟：

```powershell
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
```

>-ExecutionPolicy Bypass：將權限設為「略過 (Bypass)」，也就是不檢查任何限制。
-Scope Process：將這個設定的範圍限制在「目前的進程 (Process)」。一旦你關閉這個 PowerShell 視窗，設定就會自動失效，不會影響電腦的安全性。
